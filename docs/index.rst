API позволяет интегрировать сервис отчетности с контролирующими органами в ваши процессы и информационные системы.

Контур.Экстерн является сервисом отчетности, который обеспечивает `электронный документооборот`_ между хозяйствующими субъектами и контролирующими органами. Веб сервис обладает рядом преимуществ, но не всегда может поддержать уникальные сценарии пользователя. API Контур.Экстерна нужен, чтобы встроить работу с электронным документооборотом отчетности в интерфейсы информационных систем клиента.

Интеграционные возможности 
==========================

1. Отправка отчетности в контролирующие органы, такие как ФНС, ФСС, ПФР, Росстат.
2. Централизованный мониторинг отчетности для любой организационной структуры.
3. Автоматизация работы с требованиями – массовая загрузка поступивших требований в информационную систему клиента, формирование ответа на требование без ограничения на количество и объем отправляемых документов.
4. Организация зашифрованного или расшифрованного архива отчетов в собственной информационной системе или локальном хранилище.

Абонент Контур.Экстерна
-----------------------

Организация, которая автоматизирует процессы компании и предоставляет бухгалтерам работу с отчетностью в привычных интерфейсах корпоративной информационной системы.

Использование API Контур.Экстерн решает следующие задачи:

   * контроль процессов по работе с отчетностью для организаций с большим количеством ЮЛ в организационной структуре;
   * оптимизация работы с отчетностью большого количества пользователей;
   * полная или частичная автоматизация работы с потоком новых требований ФНС;
   * отправка большого количества документов из корпоративной информационной системы в ответ на требования ФНС;
   * организация локального архива отправленной отчетности, входящих писем и требований ФНС.

Компания-партнер Контур.Экстерна
------------------------

Компания, которая разрабатывает или предоставляет своим клиентам различные информационные системы и хочет внедрить в них электронный документооборот с контролирующими органами. Таким образом, вендору, который расширяет функциональность продукта в части электронного документооборота с контролирующими органами, не нужно самостоятельно получать статус оператора электронного документооборота.
