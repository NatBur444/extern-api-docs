DraftsBuilder
=============

   :maxdepth: 2
   :hidden:

Для двух видов документооборота (ответ на требование и регистрация бизнеса) налогоплательщик при представлении электронных документов в налоговый орган должен соблюдать требвания к составу отправляемых документов. Также файлы документов должны быть приведены к определенному виду, который устанавливается в соответствующих нормативных документах. В составе документов должен быть файл описи, в котором перечислены документы пользователя. Чтобы облегчить процесс формирования черновика для этих двух видов документооборотов мы разработали конструктор DraftsBuilder. 

DraftsBuilder приводит файлы пользователя к установленному формату, собирает документы в черновик, формирует XML-файл описи к нему. Разработанные методы облегчат работу с файлами пользователей. После использования DraftsBuilder вы продолжите работу с методами для черновиков. Таким образом, в данном разделе вы узнаете больше о DraftsBuilder, его сущностях и последовательности работы с его методами. 

Если у вас все файлы пользователя уже приведены к нужному формату и есть опись, вы можете не работать с DraftsBuilder, а сразу создавать черновик документооборота.
