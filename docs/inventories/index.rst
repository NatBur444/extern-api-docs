В предыдущих разделах мы уже писали о :doc:`категориях документооборотов</dc/категории ДО>` и знакомили вас с понятием входящего документооборота. Одним из примеров является сценарий :doc:`получения требования</knowledge base/demand>` и :doc:`ответ на него</knowledge base/inventory>`.

По законодательству на некоторые требования необходимо отвечать в электронном виде с приложением файлов и их описанием. Чтобы соответствовать этим требованиям, бал реализован новый тип документооборота — опись (ответ на требование). После отправки ответа на требования, документооборот нужно отслеживать, обрабатывать ответные документы. В данном разделе вы найдете описание методов API Контур.Экстерна, которые позволяют работать с ответом на требование.
