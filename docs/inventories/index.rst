В этом разделе расскажем, как строится процесс работы с требованиями, объясним, что такое ответ на требование в виде описи, как его собрать и отследить документооборот.

В предыдущих разделах мы уже писали о :doc:`категориях документооборотов</dc/категории ДО>` и знакомили вас с понятием входящего документооборота. Одним из примеров является сценарий получения требования и ответ на него.

По законодательству на некоторые требования необходимо отвечать в электронном виде с приложением файлов и их описанием. Чтобы соответствовать этим требованиям, мы реализовали новый тип документооборота — опись (ответ на требование). После отправки ответа на требования, документооборот нужно отслеживать, обрабатывать ответные документы. В данном разделе вы найдете описание методов API Контур.Экстерна, которые позволяют это реализовать.
