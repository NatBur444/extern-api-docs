В этом разделе расскажем, как строится процесс работы с требованиями, объясним, что такое опись, как ее собрать и отследить документооборот.

В предыдущих разделах мы уже писали о :doc:`категориях документооборотов</dc/категории ДО>` и знакомили вас с понятием входящего документооборота. Одним из примеров является сценарий получения требования и ответ на него.

По законодательству на некоторые требования необходимо отвечать в электронном виде с приложением файлов и их описанием. Чтобы соответствовать этим требованиям, мы реализовали новый тип документа — опись, а также инструмент для сбора ее в черновик — DraftsBuilder. Его методы позволяют облегчить работу с файлами пользователей, чтобы черновик описи соответствовал заданным стандартам. После отправки ответа на требования, его нужно отслеживать, обрабатывать ответные документы. В данном разделе вы найдете описание методов API Контур.Экстерна, которые позволяют это реализовать.
