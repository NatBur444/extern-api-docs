.. important:: Методы данного раздела позволяют работать только с Контур.Сертификатами электронной подписи, выпущенными Удостоверяющим центром АО "ПФ "СКБ Контур" (https://ca.kontur.ru).

Данный раздел раскрывает возможности API по работе с Контур.Сертификатом. Отличительная особенность работы с Контур.Сертификатом - это возможность пользователя разрешать его использование посредством ввода кода, полученного в смс-сообщении на номер телефона из заявления на выдачу сертификата. 

Таким образом, данный вид криптографии удобна тем, что существенно снижаются требования к программному обеспечению на рабочем месте пользователя. Нет необходимости в установке и настройке криптопровайдеров. Нужно только в момент подписания или расшифровки документа ввести код из полученного смс-сообщения.

**Алгоритм работы с Контур.Сертификат**

----------

.. image:: /_static/cloud.jpg

* передаются документы для совершения криптографических операций,
* в ответе возвращается идентификатор запроса и отправляется смс-сообщение с кодом,
* отдельным запросом передается пара идентификатор запроса + код из смс-сообщения,
* в ответе получаем результат криптографической операции.

.. note:: 
   1. Подписать некоторые технологические документы в ходе документооборота можно без ввода кода из смс-сообщения. Для этого в отдельных методах для работы с документооборотами есть специальный параметр :ref:`→ <rst-markup-post-sign-reply-doc>`.
   2. На тестовом стенде для получения кода необходимо воспользоваться telegram-ботом :guilabel:`&@cloud_crypt_sms_bot`
